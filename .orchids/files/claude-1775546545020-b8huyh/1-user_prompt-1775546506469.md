Build a full-stack remote desktop application similar to AnyDesk and RustDesk with the following requirements:

1. 🎯 CORE FUNCTIONALITY

* A "Host" user can share their screen in real-time
* A "Client" user can connect to the host using a unique session ID or code
* The client should be able to:

  * View the host screen (live streaming)
  * Control mouse and keyboard input remotely
* Low latency communication (near real-time)

2. 🖥️ FRONTEND (React)

* Use React (latest stable version)
* Create two main interfaces:

  * Host Dashboard
  * Client Connect Screen
* Features:

  * Generate session ID for host
  * Input session ID for client
  * Display live screen stream (WebRTC)
  * Capture and send user input (mouse/keyboard)
* Use clean UI (Tailwind or simple CSS)

3. ⚙️ BACKEND (Dockerized Only)

* Backend must run fully inside Docker (no local dependency required)
* Use Node.js (Express or Fastify) OR Go (preferred for performance)
* Responsibilities:

  * Signaling server for WebRTC (using WebSocket)
  * Session management (host-client pairing)
  * Authentication (optional basic token system)
* Provide:

  * Dockerfile
  * docker-compose.yml
* No backend should run outside Docker

4. 🔗 REAL-TIME COMMUNICATION

* Use WebRTC for:

  * Screen streaming
  * Data channel (for mouse/keyboard control)
* Use WebSocket (via backend) only for signaling

5. 🍎 macOS SUPPORT (IMPORTANT)

* Provide a native host agent script for macOS:

  * Use Node.js + native APIs OR Python (PyObjC) OR Swift CLI
* Capabilities:

  * Capture screen (macOS screen recording permission required)
  * Capture input events
  * Apply remote control inputs
* Include:

  * Setup script for macOS permissions
  * Instructions for enabling:

    * Screen Recording
    * Accessibility (for control)

6. 🔐 SECURITY

* Use secure WebRTC (STUN/TURN support)
* Optional:

  * Add TURN server config (coturn via Docker)
* Use environment variables for:

  * TURN credentials
  * API URLs
  * Secrets

7. 🌍 CENTRALIZED CONFIGURATION

* All configs must be controlled via `.env` files
* Examples:

  * FRONTEND_URL
  * BACKEND_URL
  * WS_URL
  * TURN_SERVER
  * TURN_USERNAME
  * TURN_PASSWORD
* Changing env variables should make the app production-ready without code changes

8. 📦 PROJECT STRUCTURE

* /client → React app
* /server → Backend (Dockerized)
* /mac-agent → macOS host controller
* /docker → docker-compose + configs

9. 🚀 DEPLOYMENT

* Provide:

  * docker-compose for production
  * Instructions for deploying on VPS
* Frontend should connect to backend via env-based URL

10. 📄 DOCUMENTATION

* Setup guide (step-by-step)
* macOS permissions guide
* How host/client connect
* How to run locally and production

Goal:
A fully working remote desktop system using WebRTC, with Dockerized backend, React frontend, macOS host support, and centralized environment-based configuration for easy production deployment.
